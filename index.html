Project Overview
Languages: HTML/CSS (Framework: Bootstrap)
Tools: Figma, IntelliJ
To redesign a webpage, we need several steps:
  1. Identifying problems that existed in the original webpage.
  2. Draw Low-Fidelity wireframes for the webpage redesign by hand. In another word, create a “draft” for the webpage redesign.
  3. Create a visual design style guide for the webpage, to identify which style we are going to use for the webpage redesign.
  4. Do High-Fidelity Prototyping, which is based on the “draft” we have to create the webpage prototype using Figma (without real coding).
  5. Build the real webpage!

Conclusion
In this project, we go through the process that redesigning a webpage needed. First, we need to find the problems of the original webpage, then draw a “draft” that seems can solve the problems. After that, draw a visual style guide and use software like Figma to build a prototype of the redesigned webpage. If it looks good, write code and build the real webpage.
At the same time, this project also explored responsive design. As the annotation in the hi-fi prototype said, the technique used to deal with the responsive functionality is the Bootstrap grid system.
